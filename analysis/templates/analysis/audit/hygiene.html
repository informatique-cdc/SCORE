Hygiène du corpus

{{ result.score|floatformat:0 }}/100

Évalue la propreté de votre corpus documentaire : doublons exacts et quasi-doublons, contenu répétitif (boilerplate), homogénéité linguistique et présence de données sensibles (PII). Un score élevé indique un corpus propre, prêt pour l'indexation RAG.

{{ metrics.sub_scores.uniqueness|floatformat:0 }}
Unicité
Proportion de chunks uniques (sans doublons exacts). Un score bas signifie que de nombreux chunks sont dupliqués.
{{ metrics.sub_scores.neardup|floatformat:0 }}
Quasi-doublons
Mesure l'absence de paires quasi-identiques (similarité élevée sans être exacte). Un score bas révèle de la redondance.
{{ metrics.sub_scores.boilerplate|floatformat:0 }}
Boilerplate
Absence de contenu répétitif générique (en-têtes, pieds de page, mentions légales). Un score bas indique du bruit dans l'index.
{{ metrics.sub_scores.language|floatformat:0 }}
Langue
Homogénéité linguistique du corpus. Un score bas signifie un mélange de langues qui peut dégrader la recherche.
{{ metrics.sub_scores.pii|floatformat:0 }}
Données sensibles
Absence de données personnelles (emails, téléphones, numéros de sécurité sociale, clés API). Un score bas requiert une action immédiate.
{{ metrics.total_chunks }}
Chunks
Nombre total de fragments de texte analysés dans le corpus.
{{ metrics.exact_duplicates }}
Doublons exacts
Chunks dont le contenu est strictement identique à un autre. À supprimer pour réduire le bruit.
{{ metrics.neardup_pairs }}
Paires near-dup
Paires de chunks très similaires (même contenu avec des variations mineures). Peuvent fausser les résultats RAG.
{{ metrics.pii_findings_count }}
PII/secrets
Occurrences de données sensibles détectées (emails, téléphones, clés API). Risque de fuite si exposé via RAG.
Distribution des longueurs

Répartition de la taille des chunks (en caractères). Une distribution trop large ou des chunks très courts/longs dégradent la qualité de la recherche.

Répartition des langues

Langues détectées dans les chunks. Un corpus multilingue non voulu peut dégrader l'embedding et la recherche sémantique.

PII par type

Répartition des données personnelles détectées par catégorie. Ces éléments devraient être anonymisés ou supprimés avant indexation.

Doublons par document

Nombre de chunks dupliqués par document source. Les documents avec beaucoup de doublons sont prioritaires pour un nettoyage.
